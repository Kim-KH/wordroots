========================================
해야 할 작업 순서
========================================

[Step 1] 즉시: 캐시 클리어 후 앱 실행 (without 번역 반영 확인)
  node scripts/build-etymology-data.js
  npx expo start --clear
  → without 일본어/중국어 번역이 표시되면 정상

[Step 2] fix-english-fields.js 동작 확인
  $env:ANTHROPIC_API_KEY = "sk-ant-..."
  node scripts/fix-english-fields.js
  → 실행 시 터미널에서 각 단어 처리 결과 확인:
    "  [1/N] 단어명... ✓"  이면 정상 저장됨
    "  [1/N] 단어명... ✗"  이면 API 오류 → 원인 파악 필요

[Step 3] 스크립트 완료 후 검증
  node -e "
  const fs=require('fs'),path=require('path');
  const KOREAN=/[가-힣]/;
  function files(d){const r=[];for(const e of fs.readdirSync(d)){const f=path.join(d,e);if(fs.statSync(f).isDirectory())r.push(...files(f));else if(e.endsWith('.json'))r.push(f);}return r;}
  let w=0,a=0;
  for(const p of ['Part_Origin','Part_Prefixes','Part_Roots','Part_Suffixes']){
    for(const f of files('src/data/'+p)){
      const d=JSON.parse(fs.readFileSync(f,'utf8'));
      for(const word of(d.words||[])){
        if(word.meaning?.en&&KOREAN.test(word.meaning.en))w++;
        for(const an of(word.analysis||[]))if(an.meaning?.en&&KOREAN.test(an.meaning.en))a++;
      }
    }
  }
  console.log('word.meaning.en 한국어 잔존:',w);
  console.log('analysis.meaning.en 한국어 잔존:',a);
  console.log(w===0&&a===0?'완료':'미완료');
  "
  → 둘 다 0 이 목표

[Step 4] 최종 빌드 및 앱 재시작
  node scripts/build-etymology-data.js
  npx expo start --clear


========================================
WordCard.js 번역 표시 로직
========================================

getTranslation() 에서 isLikelySentence() 로 필터링함:
  - 문장부호(. ! ? 。！？) 있으면 표시
  - 공백 기준 6단어 이상이면 표시
  - 둘 다 아니면 표시 안 함 (짧은 키워드로 판단)

주의: 일본어/중국어는 공백이 없으므로 반드시 문장부호(。)로 끝나야 함.
      마침표 없는 번역 → 앱에서 보이지 않음.
      예) "あなたなしでは生きられません" → 표시 안 됨
          "あなたなしでは生きられません。" → 표시됨
